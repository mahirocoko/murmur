## Key Code Changes

- Audio-reactive waveform now comes from native CPAL input rather than CSS fake animation.
- `audio-level` events are throttled to roughly every 70ms to avoid flooding the frontend.
- The indicator window is explicitly resized before showing, avoiding stale config when dev/build state lags.
- The indicator root sets `html`, `body`, and `#root` to transparent + `overflow: hidden` only for the indicator window.
- The visual model now follows the reference direction: large dark HUD, no title text, waveform as the primary signal, and bottom controls.

## Architecture Decisions

- Keep waveform state inside `IndicatorWindow`; no global app state was introduced.
- Keep native audio level derivation near `write_input_data` because that is the only reliable place already receiving live microphone samples during shortcut recording.
- Use a transparent Tauri window rather than trying to hide background with CSS alone; CSS cannot remove the native webview/window background when the Tauri window itself is opaque.
- Match the reference by changing the window dimensions rather than cramming a reference HUD into the old 260x64 pill.

## Honest Feedback

Three friction points stood out. First, I initially under-read the screenshot as a minor layout bug instead of a broader design mismatch; for UI work, the screenshot is the spec. Second, after Gemini tuned the existing pill, I reported it too positively even though the visual direction was still likely wrong. Third, the white-background issue was a native window configuration problem, not just CSS. I should have checked `tauri.conf.json` earlier because a transparent visual target requires `transparent: true`. The recovery was good, but it took one user correction too many. Next time, when Mahiro gives a visual reference, I should identify the real invariants immediately: window size, transparency, rounded panel, composition, and interaction copy.

## Lessons Learned

- For floating HUDs in Tauri, CSS transparency is not enough; the Tauri window must be transparent too.
- If a visual reference is provided, preserve its composition first, then tune details. Do not keep the old component shape just because it exists.
- Audio-reactive UI needs native live-level data; fake CSS equalizer animation is misleading when the user expects speech response.
- Direct CLI output should be reviewed against product taste; compile success is not visual acceptance.
