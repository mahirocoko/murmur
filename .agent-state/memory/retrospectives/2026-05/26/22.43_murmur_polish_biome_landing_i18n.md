## Lessons Learned

- Landing page Thai copy is not “short UI strings” once it contains hero/feature/privacy/install paragraphs. Use `kien-thai` before drafting or reviewing, not after being challenged.
- For visual motion requests, phrase verification honestly: build/lint prove the implementation compiles; animation quality still needs visual/manual QA.
- Biome introduction is intentionally noisy the first time. Report that `pnpm lint`/`pnpm format` are write-mode and may touch unrelated-looking JS/JSON/CSS formatting.
- In React files with section comments, place comments where they reflect actual order (`_Ref`, `_State`, `_Callback`, `_Memo`, `_Effect`) and avoid decorative comments that do not improve scanning.
- For indicator loading wave, a convincing loading motion needs per-bar phase variation, not only one shared keyframe with staggered positive delay.
