## AI Diary

This feature felt like a practical next step for Murmur because it touches the real daily dictation flow: people often have AirPods, iPhone Continuity mic, and MacBook microphone all visible at once, and the default can change underneath them. I tried to keep the implementation close to the current architecture instead of adding a new settings framework. Rust already owns CPAL recording and native preferences, so it made sense to keep input device selection there and let React/tray send the selected name. The UI part needed restraint: Mahiro showed Superwhisper-style menus, but the goal was not to clone every visual detail. A simple toolbar dropdown and tray submenu with checkmarks is enough. The icon handling is deliberately heuristic from the device name, not a new asset system. I also had to pay attention to synchronization: choosing from tray should update main, and choosing from main should refresh tray checkmarks. One unresolved risk is that device names are not stable IDs; CPAL gives names easily, but hardware can rename/disappear. For this early Murmur stage, name-based selection is acceptable, but I should report that manual QA matters. This is another Tauri/native feature where build passing does not prove the real mic selected during recording.

## Honest Feedback

1. The implementation uses input device names as identifiers. That is simple and probably fine for alpha, but it can be fragile if two devices share a name or macOS renames Continuity devices. I should call this out instead of pretending it is a perfect device ID system.
2. Tray menu refresh is functional but not deeply proven. Rebuilding the menu after preference changes should work, but native tray behavior on macOS needs manual verification. Compile checks do not prove checkmarks update visually.
3. The main window dropdown is intentionally small, but it lives in a tight toolbar. It may need visual QA for long device names like “Piyawat’s AirPods Pro (Default)” and for popover placement near the right edge.
4. I added icon inference by string matching. That is a useful polish layer, but it is heuristic. It should remain quiet and not become a brittle taxonomy of every hardware brand.

## Lessons Learned

- For microphone selection, preserve `Use system default` as a first-class option. Pinning a named device and following macOS default are different user intents.
- Tray and main window preferences need event-based synchronization both ways; otherwise the app feels inconsistent.
- CPAL device names are a good alpha-level key but not a perfect long-term identity. Future hardening may need stable host/device metadata if available.
- Icon polish should stay heuristic and restrained: headphones/AirPods, phone, laptop, generic mic are enough for now.
- Native audio features require manual QA: selected device must actually be used by the recording stream.
